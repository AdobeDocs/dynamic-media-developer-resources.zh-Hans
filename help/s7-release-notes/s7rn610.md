## Scene7 IPS（映像生产系统）4.5.0 {#section-3e37ed37b6d146419f4578bd7ffeaa74}

**新增功能**

* 现在正在运行JDK 1.8。
* 支持EXIF方向标签。
* 为临时FTP文件（使用ProFTP创建）上载目录文件跳过。

**不再支持的功能**

* “IPS经典发布验证”中的“目录服务器重新同步”选项 — 已删除。
* 您不能再设置空或重复元数据标记字段值。

**错误修复**

* `<metadataArray>`对于[!DNL .vtt]文件为空，以响应API `getAssets`。

* 方向缺少用户数据字段。
* `getJobLogDetails`对于更新ACO文件的作业的响应未正确指定作业详细信息。
* PSD主文件已记录为已发布的次数。
* 名称中包含无效字符的资源可上传到IPS。
* 未能生成的视频缩略图保留为“在队列中待定”状态。
* 在大`deleteFolder`和服务器目录上载之间进行文件夹还原的竞态条件。
* 在上传时提前发布时，每个发布服务器会以两次发布的方式记录PDF文件。
* 上传图层名称相同的多个PSD文件会导致优化错误。
* 客户PDF的PDF栅格化导致生成的图像出现意外的白线。
* 视频缩略图文件在上传时已多次发布。
* 重复上传文件检测生成了两个上传通知，导致丢弃完成的文件记帐。
* 视频转码的附属XML文件在更新时多次记录为已发布。
* moveFolder在删除嵌套文件夹并将另一个文件夹（不在嵌套文件夹的父文件夹中，并且与嵌套文件夹同名）移动到嵌套文件夹之前所在的位置时失败。
* 共享配置和本地配置中的群集状态配置条目未通过清除清除清除。
* 更改API日志分析器以筛选电子邮件地址格式。
* 视频处理未正确映射`Iptc4xmpCore:Location`。
* 清除和上载之间存在竞争条件。
* 发布不使用`assetFile`，因此暂存发布仅适用于新资产。
* 更新规则集时引发空指针。

## 查看器 {#section-833ab92c91c941d2bfdc27f233f582ad}

**Dynamic Media Image Serving Viewer 5.2.2的新增功能、增强和错误修复**

查看器升级向后兼容，无需更改现有Web代码。 但是，建议您在我们的暂存环境上测试新查看器。 以下网站说明了如何设置系统以访问我们的暂存服务器：

[http://helpx.adobe.com/experience-manager/scene7/kb/base/upgrade-management/testing-scene7-viewers-upgrade.html](http://helpx.adobe.com/experience-manager/scene7/kb/base/upgrade-management/testing-scene7-viewers-upgrade.html)

将计算机设置为访问暂存服务器后，可以检查网站以测试升级。 如果您使用现成的HTML5查看器，最佳实践是针对我们的标准暂存服务器和s7is5-预览-staging.scene7.com进行测试。

新增功能和增强功能

另请参阅[查看器参考指南](https://experienceleague.adobe.com/docs/dynamic-media-developer-resources/library/home.html)。

* 增加了在eCatalog查看器中对打印、下载和收藏夹的支持。
* 添加了使用getComponent API检索ParameterManager的功能。
* 转换的旋转查看器、缩放查看器、视频查看器和弹出查看器可对图稿使用精灵。
* 增加了对IE11本机全屏的支持。
* 在容器中重构了模拟（非本机）全屏支持。
* 增加CSS小标记大小以支持更大的手机。
* 删除了桌面浏览器上HTML5旋转、基本缩放、缩放、旋转和混合媒体查看器的CSS大小标记。
* 添加了支持，允许在SpinView中对预加载的帧进行质量配置。
* 固定 — Galaxy S4，纵向模式：启用全屏模式时使用的CSS大小标记不正确。
* 已修复 — 在Internet Explorer 9上使用用于外部链接的javascript模板后，将卡住[ eCatalog]图像映射。
* 已修复 — Internet Explorer 9和Internet Explorer 10:如果为CSS中的IMG定义了width属性，则缩放工作会不正确。

常规

* 修复了s7sdk.browser.device.version检测到第五个Android不正确的问题。
* 删除了可见性的用法：SDK组件中的可见样式。

浮动缩放视图

* 已修复 — [IE9]弹出窗口有时在鼠标移开后仍保留在屏幕上。
* 已修复 — 已向初始图像添加两次iscommand字符串。
* 添加了其他状态事件。

PageView

* 已修复 — 如果拖动手势垂直开始，然后水平移动，翻页功能将无法正确工作。
* 添加了其他状态事件。

## 不再支持{#section-9295ea9313c84695b19bca0e0f01a690}的功能

* **Flash查看器生命周期终止** *通知自2017年1月31日起，Adobe Scene7(现为Dynamic Media Classic)正式终止对Flash查看器平台的支持。*

* 提醒：自2014年1月31日起，Scene7正式终止了对DHTML查看器平台的支持。

   有关此更改的更多信息，请参阅以下常见问题解答网站：

   [http://dev.day.com/content/docs/en/cq/current/administering/integrating_with_omniture/scene7/dhtml-viewer-endoflifefaqs.html](http://dev.day.com/content/docs/en/cq/current/administering/integrating_with_omniture/scene7/dhtml-viewer-endoflifefaqs.html)

* Scene7 Creative Suite扩展在此版本中已弃用，因为Creative Cloud应用程序不支持Flash面板。 内容摄取工作流的进一步开发侧重于Adobe Experience Manager Dynamic Media和Creative Cloud。 对于Scene7,Scene7 Publishing System中的上载和导出功能仍保留并可改为使用。
* 由于采用率较低，UGC视频在此版本中已弃用。 设置另一个Dynamic Media Classic帐户可用作UGC视频的替代解决方案。 要设置其他帐户，请与Adobe技术支持联系。
* 由于采用率较低，此版本中已弃用文本连字。 连字词典定义了如何拆分单词和连字以支持最佳文本排列。
* 元数据服务器和相似性搜索在2015年都已弃用，因为采用率较低。 在公布正式停用目标日期后，受影响的客户将会收到Adobe技术支持的通知。
